DataSet Validation

RDO.AspNetCore supports DataSet validation, both server side and client side.

Server Side

DataSetValidatorProvider which provides custom DataSet model validator, is registered with ASP.Net Core MVC framework. All RDO.Data validation errors will be merged into ASP.Net Core model state automatically.

In addition, you can use ValidationMessageTagHelper to show error message in the view:

<span dataset-validation-for="Movie" dataset-column="@_.Title" class="text-danger"></span>

Client Side

Client side validation is done by generating data- HTML attributes that work with a custom jQuery Validate adapter. The following client side validators are provided by RDO.AspNetCore:

Validator Description
MaxLengthClientValidator Client validator for MaxLengthAttribute.
NumericClientValidator Client validator for numeric column.
RegularExpressionClientValidator Client validator for RegularExpressionAttribute.
RequiredClientValidator Client validator for RequiredAttribute.
StringLengthClientValidator Client validator for StringLengthAttribute.

These attributes are registered automatically with ASP.Net Core framework.

You can also create your own custom client side validation, by:

  1. Create your custom client validator class that implementing IDataSetClientValidator interface, or derived from DataSetClientValidatorBase<T>.

  2. Add your custom client validator object via DataSetMvcConfiguration.DataSetClientValidators of the AddDataSetMvc call in your Startup class.
